# SMTP HTML 邮件模板与预览

## Goal

将短信 SMTP 转发从难以扫读的纯文本邮件升级为结构清晰、适配桌面与移动邮箱客户端的 HTML 邮件，并允许管理员在 Dashboard 中完整编辑共享 HTML/CSS、各邮件类型正文和 Subject，实时预览最终效果。

## Background

- 当前 SMTP 邮件由 `components/idf_push/idf_push.cpp` 生成，使用 `Content-Type: text/plain; charset=UTF-8`，正文以 base64 编码发送。
- 短信邮件当前包含发件人、本机号码、时间和多行短信内容，但这些字段在进入邮件队列前被拼接为一段纯文本。
- 来电提醒当前复用短信转发队列，使用来电号码作为发件人、`来电：号码` 作为正文；模板化后需要显式区分短信与来电类型。
- SMTP 还用于每日心跳、保号结果、定时任务结果、管理员短信命令结果和 RESET 命令结果等系统通知。
- Dashboard 的 `code/web_src/panels/push.html` 已有 SMTP 配置表单；配置读取和保存分别经过 `/config.json` 与 `/save`。
- 普通配置 NVS 分区为 20KB，另有 640KB 的 `smsdata` NVS 数据分区；多份完整 HTML 模板不能无界写入普通配置区。

## Requirements

- R1: 邮件使用共享基础 HTML/CSS 包裹类型正文，基础模板必须包含 `{content}` 插槽。
- R2: 支持五种邮件类型：短信转发、来电提醒、每日心跳、保号结果、通用系统通知。
- R3: 五类正文 HTML 片段与 Subject 模板可分别编辑、保存、预览和恢复默认；共享基础模板可完整编辑。
- R4: 短信模板支持 `{sender}`、`{receiver}`、`{timestamp}`、`{message}`；来电模板支持 `{caller}`、`{receiver}`、`{timestamp}`。
- R5: 心跳、保号和通用系统模板至少支持 `{title}`、`{message}`、`{timestamp}`；心跳额外支持 `{sms_total}`、`{free_heap}`，保号额外支持 `{action}`、`{result}`。
- R6: Subject 使用对应类型的纯文本占位符，渲染后过滤 CR/LF 并限制长度，防止邮件头注入。
- R7: 所有动态字段插入 HTML 前必须转义；短信和通知的多行正文保持换行。只有渲染完成的类型正文可以作为可信 `{content}` 插入基础模板。
- R8: SMTP 邮件使用 `multipart/alternative`，同时包含 UTF-8 纯文本和 HTML 两部分。
- R9: 默认模板采用安静、实用的浅色系统通知风格：中性背景、白色主体、清晰字段层级；短信蓝绿色、心跳绿色、保号按结果绿/红、来电琥珀色、系统通知蓝色。
- R10: 默认模板不使用远程图片、外部字体、渐变或脚本，正文宽度约 600px，窄屏自适应，并使用常见邮箱客户端兼容的保守布局和内联样式。
- R11: Dashboard 在邮件设置中提供模板管理界面，包含基础模板和五类邮件标签、Subject 输入、HTML 编辑器、占位符插入、保存、恢复默认和沙箱实时预览。
- R12: 预览使用与实际发送相同的占位符规则和代表性样例；预览文档在无脚本权限的 sandbox iframe 中运行。
- R13: 保存时拒绝缺少必需占位符、包含脚本类主动内容、UTF-8 非法或超过大小限制的模板，并返回明确错误，不覆盖上次有效配置。
- R14: 运行时模板缺失、损坏或渲染失败时，自动使用对应内置默认模板发送并记录日志，不能丢失通知。
- R15: 自定义模板覆盖存放在 `smsdata` 独立 NVS 命名空间；未保存覆盖时使用固件默认值。基础模板上限 8KB、单个正文片段 4KB、单个 Subject 256B。
- R16: 邮件发送路径只读取当前类型所需模板，不在高频配置快照或邮件队列中复制所有模板。
- R17: 现有 SMTP 服务器、端口、账号、授权码、收件地址、启用状态、转发规则和重试语义保持兼容。
- R18: 定时任务、管理员短信命令和 RESET 命令结果使用“通用系统通知”模板；来电继续遵守现有黑名单、开关和转发规则。
- R19: 模板覆盖应纳入配置备份/恢复，并在恢复出厂设置时清除；旧固件升级后无覆盖数据时直接使用内置默认模板。

## Acceptance Criteria

- [ ] AC1: 中文验证码和英文营销短信均生成字段分区明确的 HTML，正文换行不丢失。
- [ ] AC2: 邮件 MIME 为 `multipart/alternative`，纯文本与 HTML 均为 UTF-8 且可独立读取。
- [ ] AC3: 含 `<`, `>`, `&`, 引号、伪标签或占位符字样的动态内容只按文本显示，不改变模板结构或触发二次替换。
- [ ] AC4: 基础模板和五类正文/Subject 可独立保存；刷新 Dashboard 后仍存在并用于后续邮件。
- [ ] AC5: 修改某一正文不会改变其他类型；修改基础模板会一致影响五类预览和实际邮件。
- [ ] AC6: Dashboard 预览随编辑更新，样例与当前类型匹配，脚本、表单和顶层导航无法从预览影响 Dashboard。
- [ ] AC7: 缺少 `{content}`、缺少类型必要字段、包含主动内容或超限时保存失败，原有效配置保留。
- [ ] AC8: 一键恢复基础模板或任一类型默认值后，预览和后续邮件恢复到固件内置样式。
- [ ] AC9: 模板读取/渲染异常时仍发送可读的默认 HTML 和纯文本邮件，并产生可诊断日志。
- [ ] AC10: 五类 Subject 模板生效；动态字段中的换行不能产生额外邮件头。
- [ ] AC11: 来电走来电模板而非短信模板；心跳、保号和通用系统邮件分别使用正确类型。
- [ ] AC12: 旧设备升级后无需重新填写 SMTP 凭据，自动使用默认模板。
- [ ] AC13: 模板可随配置备份导出和导入，恢复出厂后覆盖被清除。
- [ ] AC14: Web 资源生成检查、Dashboard 静态预览和 ESP-IDF 固件构建通过。

## Out of Scope

- 邮件附件、邮件追踪、远程图片管理和外部字体托管。
- JavaScript、iframe、object、embed、表单提交、事件处理器或其他主动 HTML 内容。
- 条件语句、循环、用户自定义占位符或按发件人选择不同模板。
- 从 Dashboard 发送真实测试邮件；本期只提供本地渲染预览。
